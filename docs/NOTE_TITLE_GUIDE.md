# Note Title Guide

Rules for picking titles for personal notes (`src/pages/notes/`).

Applies to notes, not blog posts. Posts can be creative/evocative; notes
should be scannable and say what's inside.

## Prefer gerund + object over nominalized noun phrases

For how-to / process content, use `Verb-ing + Object` rather than a nominalized form.

- "Archiving Streams" — not "Stream Archival"
- "Analyzing Logs" — not "Log Analysis"
- "Destroying Caches" — not "Cache Destruction"

Gerund form reads as active and terse; nominalized form reads as formal/academic
and doesn't fit the site's register.

## Plural object for general practice, singular for a specific thing

- "Archiving Streams" — general how-to
- "Archiving the Zaiko Stream" — one specific target

Singular-bare ("Archiving Stream") is ambiguous — it can be read as "a stream
that archives" rather than "archiving a stream" — so avoid.

## Attributive noun modifiers are singular

When using the noun-noun compound form (not the gerund), the modifier is
singular even when the meaning is plural:

- "stream archival" ✓ not "streams archival" ✗
- "car park", "book store", "data archival"

Plural modifiers only work in lexicalized idioms ("arms race", "systems analyst").

## Noun "Archive" vs gerund "Archiving"

- "Archive" (noun) = a collection of archived things. Use when the page IS
  the collection (e.g. "DNM Archive: 2013-2015").
- "Archiving" (gerund) = the process. Use for how-to content.

Don't title a how-to page "X Archive" — readers expect a catalog and get a guide.
